O projeto de banco de dados é um elemento crucial para o sucesso na implementação da tecnologia de banco de dados em uma organização. Com o crescimento das empresas e o aumento do volume de informações a serem gerenciadas, é fundamental garantir que os requisitos de informação dos usuários sejam atendidos de forma eficiente e confiável.

Este trabalho tem como objetivo desenvolver o projeto de Controle de Tráfego Aéreo, utilizando como base os princípios e conceitos dos três modelos essenciais no processo de banco de dados: o modelo conceitual, o modelo relacional e o modelo lógico. Serão explorados os principais pontos relacionados a esses modelos, adaptando-os às necessidades específicas do controle de tráfego aéreo, visando a eficiência e a segurança das operações aéreas.

O modelo conceitual representa os conceitos do negócio e as associações existentes entre eles. Nessa fase, são identificadas as entidades, os relacionamentos e os atributos principais, além das regras de negócio que regulam essas associações. O modelo conceitual é independente da tecnologia de implementação do banco de dados e busca proporcionar uma visão geral do negócio, facilitando o entendimento entre usuários e desenvolvedores.

A etapa seguinte é a criação do modelo relacional, que representa as estruturas de dados a serem implementadas, considerando o modelo de dados escolhido (hierárquico, de rede ou relacional). O modelo relacional é derivado do modelo conceitual e apresenta entidades associativas em lugar de relacionamentos muitos-para-muitos. Nessa fase, são definidas as chaves primárias das entidades, as chaves estrangeiras entre as entidades. O modelo relacional é adequado ao padrão de nomenclatura adotado pela empresa e é documentado em um dicionário de dados.

Por fim, o modelo lógico representa a implementação do modelo relacional, levando em consideração a tecnologia de banco de dados escolhida e os requisitos não funcionais, como desempenho, disponibilidade e segurança. O modelo lógico é elaborado a partir do modelo relacional e pode variar de acordo com a tecnologia utilizada. Ele define as tabelas físicas e colunas físicas, sendo a linguagem SQL a mais comumente utilizada nessa fase.

É importante destacar que a utilização desses três modelos no projeto de banco de dados proporciona benefícios significativos para as organizações. A rapidez na manipulação e acesso à informação, a redução do esforço humano, a disponibilização da informação no tempo necessário, o controle de redundância e inconsistência de informações, o compartilhamento de dados e a aplicação automática de restrições de integridade são algumas das vantagens proporcionadas pelo uso adequado de um banco de dados bem projetado.

Em conclusão, apresentaremos um projeto completo de Controle de Tráfego Aéreo, que englobe os aspectos conceituais, relacionais e lógicos do banco de dados. O projeto será embasado nos princípios da modelagem de dados e considerará os requisitos específicos do setor, visando proporcionar um sistema confiável, seguro e eficiente para o gerenciamento do tráfego aéreo."

============== Conclusão ==================

Ao longo do desenvolvimento deste trabalho, foi possível adquirir um conjunto de habilidades e conhecimentos relevantes no processo de construção de um projeto de banco de dados para o Controle de Tráfego Aéreo. Essas habilidades adquiridas são essenciais para a criação de um sistema eficiente e adequado às necessidades específicas dessa área.

Uma das principais habilidades desenvolvidas foi a compreensão dos modelos conceitual, relacional e lógico. Por meio do estudo e aplicação desses modelos, foi possível compreender suas características distintas e a importância de cada um no desenvolvimento do sistema de banco de dados. O modelo conceitual permitiu uma visão geral do negócio e facilitou a comunicação entre usuários e desenvolvedores. O modelo relacional, por sua vez, definiu as estruturas de dados a serem implementadas, enquanto o modelo lógico representou a implementação considerando uma tecnologia específica de banco de dados.

Outra habilidade adquirida foi a capacidade de realizar a modelagem de dados de forma adequada. Através da identificação e representação das entidades, relacionamentos, atributos e restrições de integridade, foi possível organizar e estruturar as informações de maneira clara e coerente. Além disso, a aplicação da normalização até a terceira forma normal possibilitou a redução da redundância e a garantia da consistência dos dados.

Durante o desenvolvimento do projeto, foi necessário adaptar os modelos de banco de dados ao contexto específico do Controle de Tráfego Aéreo. Essa habilidade de adaptação foi fundamental para garantir que o sistema atendesse às exigências operacionais e regulatórias do setor, garantindo sua eficiência e funcionalidade.

Além disso, a utilização de tecnologias e ferramentas adequadas foi um aspecto importante no processo de implementação do projeto. A seleção das tecnologia de banco de dados, sendo essas brModelo, MySQL Workbench e MySQL Shell, mais adequada para o Controle de Tráfego Aéreo exigiu conhecimentos sobre diferentes sistemas gerenciadores de banco de dados e suas funcionalidades. O uso da linguagem SQL para definição, manipulação e controle das estruturas de dados foi essencial nesse contexto.

Por fim, o desenvolvimento desse trabalho permitiu o desenvolvimento de habilidades analíticas e a capacidade de solucionar problemas relacionados à organização e gestão das informações do Controle de Tráfego Aéreo. Através do processo de projeto de banco de dados, foi possível identificar requisitos, definir estruturas de dados apropriadas e tomar decisões eficientes.

Em conclusão, a elaboração deste trabalho proporcionou a aquisição de habilidades fundamentais no projeto de banco de dados para o Controle de Tráfego Aéreo. Essas habilidades serão valiosas para atuações futuras no desenvolvimento e gerenciamento de sistemas de banco de dados em diversos setores. O conhecimento adquirido durante esse processo será uma base sólida para a atuação profissional nessa área, contribuindo para o sucesso na implementação de projetos de banco de dados de forma eficiente e eficaz.
